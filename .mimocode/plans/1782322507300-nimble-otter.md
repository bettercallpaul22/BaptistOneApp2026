# Plan: Avatar Click to Update Profile Photo

## Summary
Enable users to click on the avatar image to update only the `avatarFileId` in the Personal Information section using the profile-completion update logic, without affecting other Personal Profile data.

## Current State
- `ProfileProgressSummary.tsx` has an upload modal triggered by clicking the camera icon
- Currently uses `updateBasicProfileThunk` to update avatar
- `UserProfileImage.tsx` is a display-only component

## Required Changes

### 1. Update `UserProfileImage.tsx` (`src/components/display/UserProfileImage.tsx`)
Add an optional `onClick` prop to make the avatar clickable:
```tsx
export interface UserProfileImageProps {
  size?: 'sm' | 'md' | 'lg' | 'xl';
  className?: string;
  onClick?: () => void;
}
```
- Wrap the avatar in a `<button>` when `onClick` is provided
- Add cursor pointer and hover effects when clickable

### 2. Update `ProfileProgressSummary.tsx` (`src/pages/profile/components/ProfileProgressSummary.tsx`)
- Replace `updateBasicProfileThunk` with `updateProfileCompletionSectionThunk`
- Pass `openUploadModal` handler to `UserProfileImage`
- Update `handleAvatarUpload` to use profile-completion update:

```tsx
import { updateProfileCompletionSectionThunk } from '@/store/thunks/profileThunk';

const handleAvatarUpload = async (fileIds: string | string[]) => {
  const avatarFileId = Array.isArray(fileIds) ? fileIds[0] : fileIds;
  setUploadError(null);
  setIsUploading(true);

  try {
    await dispatch(
      updateProfileCompletionSectionThunk({
        sectionKey: 'personalInformation',
        data: { avatarFileId },
      })
    ).unwrap();
    closeUploadModal();
  } catch (err) {
    // ... error handling
  } finally {
    setIsUploading(false);
  }
};
```

## Files to Modify
