# Remove profile-completion fetch from AppShell

## Problem

`AppShell.tsx` dispatches `fetchProfileCompletionThunk()` on every mount (it's the root layout, so every page mounts it). This causes the API call to fire even on pages that don't need profile completion data (Home, Bible, Hymnal, Church).

The `ProfilePage.tsx` already has its own `fetchProfileCompletionThunk()` call at line 39, guarded by its own `profileFetchRequested` ref.

## Plan

### 1. Remove profile-completion logic from `AppShell.tsx`

- Remove the `profileFetchInitiated` ref (line 172)
- Remove the `useEffect` that dispatches `fetchProfileCompletionThunk()` (lines 175-184)
- Remove the `profileLoading` selector (line 81 — check if still used elsewhere in AppShell first)
- Remove the import of `fetchProfileCompletionThunk` (line 25)

### 2. Verify `ProfilePage.tsx` handles it alone

`ProfilePage.tsx:32-40` already dispatches `fetchProfileCompletionThunk()` once on mount, guarded by `profileFetchRequested` ref. This is sufficient — no changes needed.

## Files to modify

- `src/layouts/AppShell.tsx` — remove profile-completion fetch logic

## Verification

1. Navigate to Home/Bible/Hymnal/Church tabs → no `profile-completion` API call in Network tab
2. Navigate to Profile tab → `profile-completion` fires once
3. Put device to sleep, wake → no `profile-completion` requests on non-profile pages
