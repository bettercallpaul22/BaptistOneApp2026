# Plan: Convention Feature — Full Implementation

## Overview
Implement the complete Convention feature in the web app, matching the RN app's functionality: 4 tabs (Programs, Publications, Announcements, Documents), program registration with dynamic forms, wallet/card payments, publication purchases, and user registration/access lists.

## New Files

### 1. `src/types/convention.ts` — All convention types
- `ConventionTabKey` = `'programs' | 'publications' | 'announcement' | 'documents'`
- `ConventionProgram`, `ConventionProgramRegistrationField`, `ConventionProgramRegistration`
- `ConventionPublication`, `ConventionPublicationAccess`
- `ConventionDocument`, `ConventionAnnouncement`
- `ConventionFileAsset`
- Response wrappers: `ConventionProgramsResponse`, `ConventionPublicationsResponse`, etc.
- Helper: `getConventionIdFromMember(member)` to extract conventionId from `member.church.association`

### 2. `src/services/convention/conventionService.ts` — API service
Methods matching RN endpoints:
- `getPrograms(conventionId, { page, limit, search })` → `GET /public/conventions/{id}/programs`
- `getPublications(conventionId, { page, limit, search })` → `GET /public/conventions/{id}/publications`
- `getAnnouncements(conventionId, { page, limit, search })` → `GET /public/conventions/{id}/broadcasts?type=ANNOUNCEMENT`
- `getDocuments(conventionId, { page, limit, search })` → `GET /private/conventions/{id}/documents`
- `registerForProgram(conventionId, programId, payload)` → `POST /private/member/conventions/{id}/programs/{id}/register`
- `getProgramRegistrations(conventionId)` → `GET /private/member/conventions/{id}/registrations`
- `purchasePublication(conventionId, publicationId, payload)` → `POST /private/member/conventions/{id}/publications/{id}/purchase`
- `getPublicationAccesses(conventionId)` → `GET /private/member/conventions/{id}/publications/accesses`

### 3. `src/store/thunks/conventionThunk.ts` — Async thunks
- `fetchConventionProgramsThunk` — paginated programs
- `fetchConventionPublicationsThunk` — paginated publications
- `fetchConventionAnnouncementsThunk` — paginated announcements
- `fetchConventionDocumentsThunk` — paginated documents
- `registerForProgramThunk` — registration with payment
- `fetchProgramRegistrationsThunk` — user's registrations
- `purchasePublicationThunk` — publication purchase
- `fetchPublicationAccessesThunk` — user's publication accesses

### 4. `src/store/slices/conventionSlice.ts` — Redux state
```
State: {
  conventionId: string | null;
  programs: { items, loading, error, meta }
  publications: { items, loading, error, meta }
  announcements: { items, loading, error, meta }
  documents: { items, loading, error, meta }
  registrations: { items, loading, error }

```

### 5. `src/pages/convention/ConventionPage.tsx` — Main hub
- Resolves `conventionId` from `member.church.association` (same path as RN)
- Fetches member account on mount if needed
- Shows `AppScrollableTabs` with 4 tabs: Programs, Publications, Announcements, Documents
- Uses `AppShell` with `mobileHeaderAddon` for tabs (matches MinistryPage pattern)
- Loading/error/empty states per tab
- Fetches convention data on mount

### 6. `src/pages/convention/tabs/ProgramsTab.tsx`
- Fetches programs via Redux thunk
- Fetches user's registrations via direct API call
- Search with 400ms debounce
- Card per program: cover image, title, attendance mode, price, venue, dates, "Register" button
- Filters out already-registered programs
- "View My Registrations" link
- Pagination (load more on scroll)

### 7. `src/pages/convention/tabs/PublicationsTab.tsx`
- Fetches publications via Redux thunk
- Fetches user's accesses via direct API call
- Card per publication: cover image, title, pricing model, price, author
- Access badge ("Purchased", "Free Access") based on purchase status
- "Get Free" button for free publications, "Buy" button for paid
- "View My Publications" link
- Search + pagination

### 8. `src/pages/convention/tabs/AnnouncementsTab.tsx`
- Fetches announcements via Redux thunk
- Filters to MEMBER-targeted announcements
- Card per announcement: title, type badge, body preview, status badge, schedule
- "View Details" button → detail page
- Search + pagination

### 9. `src/pages/convention/tabs/DocumentsTab.tsx`
- Fetches documents via Redux thunk
- Card per document: title, description, file type badge, size, date
- "Open Document" button opens URL
- Search + pagination

### 10. `src/pages/convention/ConventionRegistrationPage.tsx`
- Receives program via route state
- Dynamic form builder from `program.registrationFields`
- Supports: text, email, phone, number, textarea, select, date, time, boolean
- Attendance mode selection (ONSITE/VIRTUAL) when program supports BOTH
- Two payment options: Wallet (PIN modal) or Card (Paystack checkout)
- Wallet: calls register API with `paymentMethod: 'wallet'` and `authKey`
- Card: calls register API with provider slug, opens checkout URL
- Form validation
- Success → notification + navigate back to convention

### 11. `src/pages/convention/ConventionPublicationPaymentPage.tsx`
- Receives publication via route state
- Shows publication summary, author, pricing model
- Wallet payment flow (same pattern as ResourceCartPage)
- Calls purchase API with `paymentMethod: 'wallet'`
- Card payment with Paystack checkout
- Success → notification + navigate back

### 12. `src/pages/convention/ConventionMyRegistrationsPage.tsx`
- Lists user's program registrations
- Shows PAID/PENDING status with badges
- Receipt download link

### 13. `src/pages/convention/ConventionMyPublicationsPage.tsx`
- Lists user's publication access records
- Shows active/expired status with date checks
- Download publication button (opens URL)

### 14. `src/pages/convention/ConventionAnnouncementDetailPage.tsx`
- Shows single announcement with title, type, body, schedule, audience targets

### 15. `src/services/api/endpoints.ts`

```ts
publicConventions: {
  programs: (id, { page, limit, search }) => `/public/conventions/${id}/programs?...`,
  publications: (id, { page, limit, search }) => `/public/conventions/${id}/publications?...`,
  announcements: (id, { page, limit, search }) => `/public/conventions/${id}/broadcasts?type=ANNOUNCEMENT&...`,

privateConventions: {
  documents: (id, { page, limit, search }) => `/private/conventions/${id}/documents?...`,

  registrations: (conventionId) => `/private/member/conventions/${conventionId}/registrations`,
  purchasePublication: (conventionId, pubId) => `/private/member/conventions/${conventionId}/publications/${pubId}/purchase`,
  publicationAccesses: (conventionId) => `/private/member/conventions/${conventionId}/publications/accesses`,
},
```
