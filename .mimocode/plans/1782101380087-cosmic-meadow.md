# Plan: Disable Onboarding & Speed Up RokswoodOneMobileApp Launch

## Problem Analysis

Comparing **BaptistOneApp** (fast) vs **RokswoodOneMobileApp** (slow), there are 4 key differences causing the launch delay:

| Factor | BaptistOneApp (fast) | RokswoodOneMobileApp (slow) |
|---|---|---|
| Splash delay | None | 3-second forced wait (`MIN_SPLASH_DURATION_MS = 3000`) |
| Onboarding | None | 3-slide onboarding before WebView loads |
| Caching | `cacheEnabled` (default true) | `cacheEnabled={false}`, `cacheMode="LOAD_NO_CACHE"`, `NO_CACHE_HEADERS`, cache-busted URLs |
| Loading state | `startInLoadingState` + `renderLoading` (native) | Custom overlay with 15s timeout |

## Changes

### 1. Skip onboarding screen — `App.tsx`
- Remove the `'onboarding'` stage entirely from `AppStage` type
- Change the boot-up flow: after splash completes (or immediately), go straight to `'webview'`
- Remove the `MIN_SPLASH_DURATION_MS` 3-second delay — replace with a minimal ~500ms or remove entirely
- Remove `OnboardingScreen` import and rendering

**File:** `/Users/progressive/Documents/projects/RokswoodOneMobileApp/App.tsx`

### 2. Enable WebView caching — `AppWebView.tsx`
- Set `cacheEnabled={true}` (remove the `false`)
- Remove `cacheMode="LOAD_NO_CACHE"`
- Remove `NO_CACHE_HEADERS` from the `source` prop
- Remove the `getCacheBustedUrl` function — use `APP_URL` directly
- Remove the `MAX_LOADING_OVERLAY_MS` timeout or reduce significantly

**File:** `/Users/progressive/Documents/projects/RokswoodOneMobileApp/src/features/webview/AppWebView.tsx`
