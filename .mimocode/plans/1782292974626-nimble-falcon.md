# Plan: Show Cancel Button After Joining Ministry

## Problem
After clicking "Join" on a ministry, the UI sets `hasPendingRequest = true` but does NOT set `pendingRequest` (line 89 of ministrySlice.ts). The Cancel button at line 153 of MinistryPage.tsx requires BOTH `hasPendingRequest && pendingRequest` to render, so it never appears.

## Solution
After a successful join request, refetch church ministries so the server response populates both `hasPendingRequest` and `pendingRequest` correctly.

## Files to Modify

### 1. `src/store/thunks/ministryThunk.ts`
- Update `requestToJoinMinistryThunk`: after successful join, call `ministryService.getChurchMinistries()` and return both `ministryId` and the updated ministries list

### 2. `src/store/slices/ministrySlice.ts`
- Update `requestToJoinMinistryThunk.fulfilled` handler: replace the manual `hasPendingRequest = true` with the fresh church ministries data from the refetch

## Verification
- Click "Join" on a ministry → should show "Pending" badge and "Cancel" button
- Click "Cancel" → should revert to "Available" and "Join" button
- Refresh page → pending state should persist (server-side)
