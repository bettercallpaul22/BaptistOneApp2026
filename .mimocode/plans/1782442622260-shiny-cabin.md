# Plan: Implement NotificationsScreen in BaptistOneweb

## Goal
Port the React Native notification screen logic from `baptistone-app-reactnative` to the BaptistOneweb web app, creating a full notification list screen with filtering, pagination, mark-as-read, delete, and detail modal — styled to match the web app's existing UI and TailwindCSS patterns.

---

## Current State Analysis

### RN App (source)
- **NotificationsScreen.tsx** (932 lines): FilterDropdown, SwipeableRow (gesture-handler), FlatList with pagination, Modal detail view
- **notificationSlice.ts** (458 lines): 8 async thunks, `FilterCategory` type, pagination state, mark-read tracking, 9 filter categories

### Web App (target)
- **notificationSlice.ts** (31 lines): Toast-only push/remove — needs to be **replaced** with the full server-backed notification slice
- **Tech stack**: React 19, Redux Toolkit, TailwindCSS 4, Axios, lucide-react, React Router
- **Existing patterns**: `AppStateFeedback` for loading/error/empty states, AppModal, AppButton, etc.

---

## Implementation Plan

### Step 1: Replace `notificationSlice.ts` with full server-backed slice
**File**: `src/store/slices/notificationSlice.ts`

Replace the toast slice with the full RN-equivalent slice adapted for web:
- **Types**: `Notification`, `NotificationType`, `FilterCategory`, `NotificationMeta`, `NotificationListResponse`, `UnreadCountResponse`
- **State**: `notifications[]`, `isLoading`, `isLoadingMore`, `isRefreshing`, `error`, `currentPage`, `totalPages`, `total`, `hasMore`, `activeFilter`, `unreadCount`, `markingReadIds[]`, `isMarkingAllRead`
- **Thunks** (using Axios `api` client):
  - `fetchNotifications({ filter, limit?, isRefresh? })` → `GET /private/notifications?page=1&limit=20&unreadOnly=bool&resourceType=X`
  - `loadMoreNotifications()` → next page
  - `markNotificationAsRead({ notificationId, read })` → `PATCH /private/notifications/:id/read`
  - `markAllNotificationsAsRead()` → `PATCH /private/notifications/read-all`
  - `deleteNotification(notificationId)` → `PATCH /private/notifications/:id/read` (mark as read/delete)
  - `fetchUnreadCount()` → `GET /private/notifications/unread-count`
- **Reducers**: `setActiveFilter`, `clearNotifications`, `setUnreadCount`

**Important**: Keep the toast `pushNotification`/`removeNotification` actions as well — rename to `pushToastNotification`/`removeToastNotification` or create a separate `toastSlice.ts` to avoid breaking existing toast consumers.

### Step 2: Add API client if not already present
**File**: `src/services/api.ts` (check if exists)

Ensure an Axios instance configured with the API base URL and auth headers. The RN app uses `@shared/api/client` — need equivalent for web.

### Step 3: Create the NotificationsPage
**File**: `src/pages/notifications/NotificationsPage.tsx`

Adapt the RN screen to web (TailwindCSS, no gesture-handler, no FlatList):

**Layout** (top to bottom):
1. Header with "Notifications" title
2. Unread count badge + "Mark all read" button
3. Filter dropdown (animated with CSS transitions or Tailwind)
4. Swipe hint text
5. Notification list (scrollable div, not FlatList)
6. Empty state / error state with retry (use `AppStateFeedback`)
7. Detail modal

**Notification card** (replaces SwipeableRow):
- Icon badge (colored by type) + title + body + timestamp + channel chips
- Unread indicator (bold title vs light)
- Click opens detail modal
- Delete button (X icon) instead of swipe-to-delete (web has no gesture)
- CSS hover state for delete reveal

**Filter dropdown**:
- Trigger button showing current filter
- Animated dropdown panel with filter options
- Gold accent color for active state

**Detail modal**:
- Title, timestamp, body, channel chips
- Close button
- Uses `AppModal` pattern from the web app

### Step 4: Add route
**File**: `src/routes/` (check existing route config)

Add `/notifications` route pointing to `NotificationsPage`.

### Step 5: Add navigation entry
**File**: Wherever the main nav/menu is defined

Add notifications link with bell icon and unread count badge.

### Step 6: Connect to socket (optional enhancement)
The web app already has `realtimeNotificationSlice` for socket events. Wire `notification.created` socket events to trigger `fetchNotifications()` or prepend to the list.
