# Wallet PIN Setup Modal for Giving

## Problem
When giving via wallet fails with "authKey is required for wallet giving", the user needs to set up a PIN first. Show a pin setup modal, then retry the giving after PIN is set.

## API
- **Endpoint**: `PUT /private/wallets/auth`
- **Payload**: `{ authKey: string }` (the 4-digit pin as string)
- **Response**: `{ status: boolean; message: string }`

## Files to Modify

### 1. `src/services/wallet/walletService.ts`
Add `setWalletPin(authKey: string)` method:
```ts
setWalletPin: async (authKey: string) => {
  return http.put<ApiResponse<{ status: boolean; message: string }>, { authKey: string }>(
    '/private/wallets/auth',
    { authKey },
  );
},
```

### 2. `src/store/thunks/walletThunk.ts`
Add `setWalletPinThunk({ authKey })`:
```ts
export const setWalletPinThunk = createAsyncThunk<
  { status: boolean; message: string },
  { authKey: string },
  { rejectValue: ReturnType<typeof toApiError> }
>('wallet/setWalletPin', async ({ authKey }, { rejectWithValue }) => { ... });
```

### 3. `src/pages/giving/GivingPage.tsx`

**State additions:**
- `isPinModalOpen: boolean`
- `pinValue: string` (4-digit pin)
- `confirmPinValue: string` (4-digit confirm)
- `pinError: string`
- `settingPin: boolean`

**Error interception in handleSubmit:**
When `handleSubmit('wallet')` catches an error with message containing "authKey is required":
- Open the pin modal instead of showing notification
- Store the pending giving params to retry after pin is set

**Pin Modal UI:**
- `AppModal` with title "Set Wallet PIN"
- Two `AppInput` fields (type="password", inputMode="numeric", maxLength=4):
  - "Enter PIN" — 4 digits
  - "Confirm PIN" — 4 digits
- Validation: both must be 4 digits and match
- Error text if mismatch
- Footer: Cancel + "Set PIN" button (gold/secondary variant)
- On PIN set success → close modal, retry wallet giving with same params

## Verification
- Click "Give via Wallet" without PIN set → error triggers → pin modal opens
- Enter mismatched pins → error shown, button disabled
- Enter matching 4-digit pins → "Set PIN" enabled
- Set PIN succeeds → modal closes, wallet giving retries automatically
- `npx tsc --noEmit` passes
