# Notification Socket Listener Implementation Plan

## Overview
Port the real-time notification socket listener from the React Native app (`NotificationSocketListener.tsx`) to the web app, adapted for web patterns (no local push notifications, uses browser APIs).

## What the RN Socket Listener Does
1. Connects to `SOCKET_NOTIFICATIONS_URL` via socket.io with auth token
2. Listens for 3 events:
   - `notification.created` — adds/updates notification in list, refreshes member data on member-related types, refreshes wallets on transaction types
   - `notification.unread-count.updated` — updates unread badge count
   - `notification.read-state.updated` — marks notification as read/unread in list
3. Handles reconnection with exponential backoff on server disconnects

## Changes Required

### 1. Install `socket.io-client`
```bash
npm install socket.io-client
```

### 2. Add environment variable
**File:** `.env` + `src/config/env.ts` + `src/vite-env.d.ts`
- Add `VITE_SOCKET_NOTIFICATIONS_URL` (same base as API + `/notifications` path)
- Expose via `env` config object
- Add TypeScript type for `ImportMetaEnv`

### 3. Create real-time notification types
**New file:** `src/types/realtimeNotification.ts`
- `RealtimeNotification` interface (id, type, title, body, resourceType, readAt, data, etc.)
- Mirror the RN `Notification` type structure

### 4. Create real-time notification slice
**New file:** `src/store/slices/realtimeNotificationSlice.ts`
- State: `notifications[]`, `unreadCount`, `activeFilter`
- Reducers: `setNotifications`, `setUnreadCount`, `setActiveFilter`
- Keep existing `notificationSlice.ts` untouched (it's for toast UI)

### 5. Register slice in root reducer
**File:** `src/store/rootReducer.ts`
- Add `realtimeNotification` reducer

### 6. Create `useNotificationSocket` hook
**New file:** `src/hooks/useNotificationSocket.ts`
- Accepts: `accessToken`, `notifications`, `activeFilter`, `dispatch`, optional `onNotificationCreated` callback
- Creates socket.io connection with auth, reconnection config
- Registers listeners for the 3 socket events
- Handles server-disconnect reconnection with rate limiting (same logic as RN)
- Returns socket ref
- Cleanup on unmount or token change

### 7. Create `NotificationSocketListener` component
**New file:** `src/components/realtime/NotificationSocketListener.tsx`
- Reads `accessToken` from `state.auth.accessToken`
- Reads notifications/activeFilter from `state.realtimeNotification`
- Dispatches `fetchMemberAccountThunk` on member-related notification types
- Dispatches `fetchWalletsThunk` on transaction-related notification types
- Calls `useNotificationSocket` hook
- Renders nothing (`return null`)

### 8. Mount in App.tsx
**File:** `src/App.tsx`
- Add `<NotificationSocketListener />` inside `<Provider>`, alongside `<ToastViewport />`
