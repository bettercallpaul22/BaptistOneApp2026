# Plan: Update `member.membership.approved` handler to use authAccess thunk

## Overview
When a `member.membership.approved` socket notification arrives, call `fetchAuthAccessThunk()` (which hits `GET /auth/access`) to get fresh access data, then switch to the `church-member` access if available and not already active.

## File to Modify
- **`src/hooks/useNotificationSocket.ts`** (lines 98–106)

## Change
Replace the current `member.membership.approved` block (lines 98–106):

**Before:**
```ts
if (notification.type === 'member.membership.approved' && authDataRef.current) {
  const targetAccess = authDataRef.current.userAccess.find(
    (a: UserAccess) => a.status === 'active' && a.resourceType === 'church-member',
  );
  if (targetAccess && authDataRef.current.currentAccess?.resourceType !== 'church-member') {
    console.log('[socket] switching to church-member access', { accessId: targetAccess.id });

  }
}
```

**After:**
```ts
if (notification.type === 'member.membership.approved') {
  import('@/store/thunks/authAccessThunk').then(({ fetchAuthAccessThunk }) => {
    dispatchRef.current(fetchAuthAccessThunk()).then((result) => {
      if (result && 'userAccess' in result) {
        const targetAccess = result.userAccess.find(
          (a: UserAccess) => a.status === 'active' && a.resourceType === 'church-member',
        );
        if (targetAccess && result.currentAccess?.resourceType !== 'church-member') {
          console.log('[socket] switching to church-member access', { accessId: targetAccess.id });
          dispatchRef.current(switchAccessThunk({ accessId: targetAccess.id }));
        }
      }
    });
  });
}
```

This uses dynamic import (consistent with existing pattern in this file) and reads the fresh thunk result directly rather than stale ref data.
