# Plan: Fix raw JSON in Emergency Contact modal address textarea

## Problem
When the Emergency Contact modal mounts, the address textarea shows raw JSON:
```
{
  "street": "lagos nigeria"
}
```
The API returns `emergencyContact.address` as `{ "street": "lagos nigeria" }` (nested object), but the schema defines `address` as a `textarea` expecting a flat string. `stringifyFieldValue` (line 182 in `profileFormUtils.ts`) blanket-converts any object to `JSON.stringify(value, null, 2)`.

## Root cause
`buildInitialFormValues` calls `stringifyFieldValue(getNestedValue(sectionData, field.name), field.type)`. For the `address` field, `getNestedValue` returns the object `{street: "lagos nigeria"}`, and `stringifyFieldValue` JSON.stringifies it since `typeof value === 'object'`.

## Fix

### `src/pages/profile/utils/profileFormUtils.ts`
In `buildInitialFormValues`, add logic to flatten known nested objects before passing to `stringifyFieldValue`:
- If the value is an object and the field type is `textarea` or `text`, extract string properties and join them (or extract `street` specifically for address-like fields)
- Alternative: modify `stringifyFieldValue` to handle nested objects with string-only values by joining them

The cleanest approach is to add a helper that checks if an object value has only simple string properties and joins them, rather than JSON.stringifying.

## Files to modify
- `src/pages/profile/utils/profileFormUtils.ts` — fix `stringifyFieldValue` or `buildInitialFormValues` to flatten nested objects for text/textarea fields
